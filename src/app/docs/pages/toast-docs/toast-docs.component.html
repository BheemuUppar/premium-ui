Angular Toast System

Non-blocking feedback for save confirmations, async uploads, undo actions, and system alerts. Toasts are orchestrated through PuiToastService, rendered in stacked viewports, and styled with semantic Lucide icons and subtle tinting — built for production SaaS apps.

Overview

Inject PuiToastService anywhere in your app and call a semantic method. Mount pui-toast-viewport once at the root — the service handles stacking, timers, and accessibility for you.

Use string messages for quick feedback, or pass a PuiToastInput object when you need titles, descriptions, custom icons, or action buttons in one call.

Show success toast Show object toast

Installation

Add the viewport beside your router outlet. The service is provided in root — no module imports required beyond the viewport component itself.

When to use toasts

  • Confirmations — save, copy, delete with optional undo
  • Async workflows — loading → success/error via stable ids or promise()
  • Low-attention feedback — compact and snackbar modes for transient messages
  • Rich announcements — title + description + CTA for deployments and billing events

Variants & icons

Each semantic variant maps to a Lucide-aligned outline icon: success → circle-check, error → circle-alert, warning → triangle-alert, info → info, loading → loader-circle. Override with iconName or hide icons with icon: false.

Positions

Each toast targets one of six viewport anchors. Snackbars default to bottom-center; all other variants default to bottom-right unless you pass position.
