Performance

Scalable rendering and worker processing

Premium UI separates computation from rendering. Enable virtualScroll for DOM efficiency and opt into useWorker to delegate heavy search/filter work off the main thread.

What workers handle

  • Text search, fuzzy search, keyword search
  • Filtering, sorting, grouping, ranking
  • Large dataset indexing and transformations

What workers do NOT handle

  • DOM rendering or virtual scroll (stays on main thread)
  • Overlays, focus management, keyboard navigation, animations

Datasets are initialized once inside the worker. Subsequent operations send only query payloads — not the full options array — to minimize serialization cost.

Implementation notes

  • Features initialize lazily — search UI renders only when searchable and open
  • OnPush change detection with signal-derived filtered options
  • Debounced worker search with cancellation for stale queries
  • Automatic main-thread fallback when workers are unavailable
  • Fixed itemHeight for predictable virtual scroll measurements
