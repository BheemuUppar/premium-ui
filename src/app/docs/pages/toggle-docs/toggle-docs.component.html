Overview

Toggles represent pressed/unpressed action states — formatting controls, view switchers, segmented selectors, and filter chips. They use aria-pressed and feel like a unified surface, not separate buttons.

Toggle vs Switch

  • Switch — binary on/off settings (notifications, dark mode). Uses role="switch".
  • Toggle — pressed action state (Bold, Grid view). Uses role="button" + aria-pressed.

Segmented vs Toolbar

  • Segmented (mode="segmented") — single selection with sliding active pill. Grid/List, Week/Month/Year.
  • Toolbar (mode="toolbar") — multi-select compact icon controls. Bold/Italic/Underline.

Accessibility

  • role="button" with aria-pressed="true|false" on each toggle
  • role="toolbar" on toolbar groups; role="group" otherwise
  • Roving tabindex — one tab stop per group, arrow keys move focus
  • Focus-visible ring via shared focus mixin (WCAG AA contrast)
  • aria-label required for icon-only toggles
  • aria-orientation reflects horizontal/vertical layout
  • Respects prefers-reduced-motion — indicator and hover motion disabled

Best practices

  • Use segmented mode for mutually exclusive options (view mode, time period).
  • Use toolbar mode with multiple for independent formatting actions.
  • Provide ariaLabel on every group and icon-only toggle.
  • Do not use toggles for persistent settings — use Switch instead.
