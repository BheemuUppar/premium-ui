Overview

Dialog builds on the reusable PuiOverlayService foundation. dialog.open() returns a PuiDialogRef. Pass data in the config — the dialog component reads it via injectPuiDialogData() and closes with injectPuiDialogRef().close(result). Subscribe to afterClosed() on the opener to receive the result.

Component dialog (service)

Material-style flow: an opener calls dialog.open(Component, { data }) and keeps the returned PuiDialogRef. The dialog component injects injectPuiDialogData() and injectPuiDialogRef(), then calls dialogRef.close(result). The opener receives result from ref.afterClosed().

Injection tokens

PuiDialogService provides PUI_DIALOG_DATA and PUI_DIALOG_REF to every component dialog. Use the typed helpers in dialog components — same pattern as Angular Material MAT_DIALOG_DATA / MatDialogRef.

PuiDialogRef

Returned by dialog.open(). Call close(result) from the opener or from inside the dialog via injectPuiDialogRef(). afterClosed() emits the result after the exit animation.
